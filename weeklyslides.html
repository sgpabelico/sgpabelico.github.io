Marks

Marks provide the basic shapes whose position, size, color, etc. can be encoded from data

Currently-supported marks include: area, bar, circle, line, point, rect, and text

This is not all mark types, but let's cover each of these with short examples

Marks: Area

Area charts are useful for showing changes over time

This is the sp500.csv dataset:

Marks: Area

Vega spec doc:

							"width": 300,
							"height": 200,
							"data": {
								"url": "https://vega.github.io/vega-datasets/data/sp500.csv"

							"mark": "area",

Encodings

Encoding map data fields to "channels" that correspond to position and mark properties

The most common position channels will be x, y, and color

Encodings

The encoding field defines the x, y, and color values for a mark type that accepts those inputs 

						{
							"mark": "line",
							"encoding": {
								"x": {
									"field": "date",
									"type": "temporal",
								},
								"y": {
									"field": "close",
									"type": "quantitative"
								}
							}
						};

Encodings

Encodings

Other marks use different channels:

  • geoshape marks use long and lat
  • arc marks use theta and radius
  • errorbar marks use xError and yError

Types

What kind of data is this?

There's 5 basic types

  • "quantitative" for continuous numerical values
  • "ordinal" for categorical values in some order
  • "nominal" for categorical values in no order
  • "temporal" for values of dates and times
  • "geojson" for geographic shapes

The first three will be the most frequent you use

Customizing colors

The color of the mark can be customized


						{
							...
							"mark": {
								"type": "area",
								"color": "red"
							},
							...
						};

Customizing colors

The background of an image defaults to white, but can be changed to another color


						{
							...
							"background": "red",
							"mark": "area",
							...
						};

Customizing colors

The "view" is the background of a mark and can be customized


						{
							...
							"view": {
								"fill":"red"
							},
							"mark": "area",
							...
						};

Customizing colors

The grid and the tick colors can also be customized


						{
							...
							"encoding": {
								"y": {
									...
									"axis": {
										"tickColor": "red",
										"gridColor": "red"
									}
							...
						};

Customizing size

The size of an image is defined in pixels


						{
							...
							"width": 300,
							"height": 200,
							"mark": "area",
							...
						};

Customizing size

Changing the size should re-scale the elements


						{
							...
							"width": 800,
							"height": 200,
							"mark": "area",
							...
						};

Customizing size

The visualization can also be automatically scaled to the width of the div element using "container"


						{
							...
							"width": "container",
							"height": 200,
							"mark": "area",
							...
						};

It turns out Reveal.js breaks this but a style="mid-width:100px" in the div is a work-around

Customizing font

There are a number of web safe CSS fonts


						{
							...
							"width": "container",
							"height": 200,
							"config": {
								"font":"cursive"
							},
							...
						};

Customizing font

Font color can also be customized by changing the axis object within the encoding


							...
							"encoding": {
								"y": {
									"aggregate": "count", 
									"title": "Number of Cars"
									"axis":{
										"labelColor": "red",
									}
							...

										"labelColor": "red",

Composing facets, layers, concats

The visualizations we've done to date have involved a single "view"

Vega can also "compose" multiple views together for comparing and overlaying data

These are known as "trellis", "lattice", "grid", "panel", or "facet" charts

Facets

Facets allows us to break up a visualization with multiple categories into separate visualizations

Here's an example of a histogram of the cars dataset's horsepower by country of origin
